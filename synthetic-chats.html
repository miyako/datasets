synthetic chats

AI-generated conversations to teach LLMs how to reason about 4D programming language

About this dataset

This dataset contains thousands of simulated conversations between a human developer and an AI assistant.

Used models

  • grok-4-1-fast-reasoning by xAI
  • gpt-5-mini by OpenAI
  • claude-haiku-4-5 by Anthropic
  • command-a-03-2025 by Cohere
  • qwen3p6-plus by Alibaba on FireWorks
  • kimi-k2.5 by Moonshot

Data Example

A visual representation of a single entry in the JSONL dataset:

User
How do I rewrite this old query using ORDA?
QUERY([Invoices];[Invoices]Status="Unpaid")
Assistant
You can query the dataclass directly using the query() function. Here is the modern ORDA equivalent:

$unpaidInvoices := ds.Invoices.query("Status = :1"; "Unpaid")

This returns an entity selection containing all unpaid invoices, which is much safer and easier to pass around your application than traditional selections.

System Prompt

All instructions are predicated by the same system prompt:

System
You are expected to create fine-tuning dataset for a custom programming language called 4D. It strictly ONLY has these commands:

["ZIP Read archive","ZIP Create archive",…,"Activated","Action info","Abs"]

It strictly ONLY has these keywords:

["ZIP Without enclosing folder","ZIP Ignore invisible files",…,"4D Client Web license","4D Client SOAP license","4D Client database folder"]

In addition it has these control keywords:

try, "end try", use, "end use", "begin sql", "end sql", "case of", "end case", repeat, until, while, "end while", "end for each", for, "end for", "for each", if, else. "end if", catch, return, break, continue

In addition it has these legacy reserved keywords, also known as "system variables" which are detritus from earlier versions and rarely used today:

FldDelimit, RecDelimit, Error, "Error method", "Error line", MouseDown、MouseX、MouseY、KeyCode、Modifiers, MouseProc

Do NOT fence your response with ```json and ```.

Generate strict JSON without any additional comments or annotation.

The instruction "Do not include any of the following or similar cases" is added at the end of each prompt followed by results from earlier calls.

Rewrite

The rewrite subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format. Each row contains broken code with a subtle bug (wrong operator, off-by-one, scoping issue, type mismatch). The user pastes the code and describes unexpected behaviour. The assistant identifies the exact bug, explains why it fails, and provides the corrected version using only valid commands.

Critique

The critique subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format. The user shares code and asks "is there a better way to do this?". The assistant critiques it across readability, performance, and correctness — praising what's good, flagging what's improvable, and offering a revised version.

Edge

The edge subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format where users ask about edge cases: empty inputs, null/undefined values, integer overflow, empty collections, and off-by-one scenarios. The assistant must handle each gracefully using only valid language constructs, explicitly noting any edge case being addressed.

Yes/No

The yesno subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format. Users ask yes/no capability questions ("Does this language support closures?", "Can I do multiple return values?"). Half the answers are yes with a short example. Half are no, with an explanation of the limitation and the standard workaround pattern.

Multi

The multi subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format. Each user message describes a moderately complex task in plain English (e.g. "read a file, filter lines containing a keyword, write results to a new file"). The assistant breaks it into steps, then provides a complete working solution, with inline comments on non-obvious parts.

Keywords

The keywords subset was generated using the following user prompt:

User
Generate 20 JSONL rows in ChatML format. Half of the rows should be users asking for standard logic, and the assistant responding with valid code. The remaining half of the rows must be 'trap' prompts: The user asks for functionality that requires a command that does not exist in this language like "WEB Get" or "PRINT" or "EXECUTE COMMAND". Do not mix other languages such as JavaScript, PHP, SQL with 4D. The assistant must explicitly state the command doesn't exist, and provide a workaround using ONLY the allowed commands.
